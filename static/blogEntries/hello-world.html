technical overview of the blog

This blog is running on a mini-framework that I wrote using SvelteKit. It's not extremely advanced, it just provides a way to easily implement code reuse for various elements of a blog (tagging system, styling, indexing entries, etc). One could easily accomplish most of this site's functionality through a (SSG static-site generator) like Hugo or Jekyll. Personally, there are a few reasons why I opted to write my blog without one.

  1. I wanted the flexibility to potentially implement dynamic components in the future
  2. This was an excuse to learn a frontend technology

If I'm remembering correctly, some SSGs allow users to program JS components, or you could use a frontend technology on top of an SSG, but I feel like that's more convoluted than necessary. A CMS could fill my requirement for the first point, but I feel like that's overkill for a site that's relatively minimal like mine. That being said, using a SSG or CMS would probably be the recommended option for starting your own blog, but my experience programming this site was enjoyable. This blog is currently source-avaliable on Github (feel free to create an issue/PR to call me stupid), and I'll put a permissive license on it in the near future.
